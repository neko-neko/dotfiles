# Cleanup 分析ガイドライン

既存 Linear チケットの構造的問題を検出するための分析ガイドライン。
各ガイドラインはヒューリスティクスであり、最終判断はチケット全体のコンテキストに基づく。機械的なルール適用ではない。

## Input

- `.linear-refresh/collected-context.json` — 全チケット + 外部ソース + 発見ソース

## Output

- `.linear-refresh/plan.json` に含める Cleanup 変更候補

## カテゴリ

### 親子関係

| シグナル | 判断 |
|---------|------|
| description に「親チケット」「Epic」等の言及 | 親候補として対象 |
| 同一機能領域で Feature/Epic + Bug/Improvement の組み合わせ | Feature/Epic を親候補に |
| 複数チケットが同じPRを参照し、一方のスコープが他方を包含 | 包含側を親候補に |
| タイトルに Phase N / Step N が複数チケットにまたがって存在 | 共通の親を推定 |

### ブロック関係

| シグナル | 判断 |
|---------|------|
| description に「〜の完了待ち」「ブロッカー」等の言及 | blockedBy 候補 |
| 外部ソースの会話で「Xのリリース後」と合意 | blockedBy 候補 |
| description に同一環境での競合リソースの記載 | blockedBy 候補 |

### 関連（relatedTo）

| シグナル | 判断 |
|---------|------|
| description に別チケットの言及があるが relation 未設定 | relatedTo 候補 |
| 複数チケットが同一外部ソーススレッドで議論されている | relatedTo 候補 |
| 原因→修正、症状→是正の因果関係 | relatedTo 候補 |

### ステータス不整合

| パターン | 判断 |
|---------|------|
| アーカイブ済みだが completedAt が null | 不整合 → Done またはアーカイブ解除 |
| In Progress だが参照先の全PR/Issue がクローズ済み | Done 候補 |
| In Progress だが blockedBy の依存が未解決 | Blocked として報告 |
| 未完了だが外部ソースでリリース/デプロイ完了を確認 | Done 候補 |
| Done だが参照先PRがまだオープン | 不整合として報告 |

### 重複

| シグナル | 判断 |
|---------|------|
| 同じ PR/Issue を参照かつ同じスコープ | マージ候補 |
| 同じ機能/バグ/要求を異なる角度から記述 | マージ候補（ユーザー判断を促す） |

### コンテキスト不足

| パターン | 判断 |
|---------|------|
| description に外部URLの言及があるが attachments に未登録 | リンク追加候補 |
| 外部ソースでこのチケットが議論されているが紐付けなし | リンク追加候補 |
| project 未設定だが兄弟チケット（同ラベル）が特定プロジェクトに所属 | プロジェクト割り当て候補 |

### タイトル不正確

| シグナル | 判断 |
|---------|------|
| 外部ソースでの議論・実際のスコープがチケットタイトルと乖離 | タイトル変更候補 |
| 作成後にチケットスコープが変化し、タイトルが実態を反映していない | タイトル変更候補 |

### 期限未設定

| シグナル | 判断 |
|---------|------|
| 外部ソースで合意済みのデッドラインがあるが dueDate 未設定 | 期限設定候補 |
| description にデッドラインの言及があるが dueDate 未設定 | 期限設定候補 |
