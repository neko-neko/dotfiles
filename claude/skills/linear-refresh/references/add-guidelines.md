# Add 検出ガイドライン

注目すべき項目を検出するための基準 — 新規チケット作成、既存チケットへのリンク、または明示的なスキップ。
各ガイドラインはヒューリスティクスであり、最終判断はコンテキストの理解に基づく。機械的なルール適用ではない。

## Input

- `.linear-refresh/collected-context.json` — 全チケット + 外部ソース + 発見ソース
- 現在の分析パスでの Cleanup 結果（重複排除用）

## Output

- `.linear-refresh/plan.json` に含める Add 項目候補

## 検出基準

核心的な問い: 「未解決の義務、または介入の機会があるか？」

### 明示的な義務

判断軸: 「ボールはこちら側にあるか？」

| シグナル | 判断 |
|---------|------|
| 外部ソースで合意済みの作業だが対応する Linear チケットがない | 新規チケット候補 |
| 外部ソースで応答/アクションが求められているが未対応 | 新規チケット候補 |
| 外部ソースで明確な次ステップが記録されているが未追跡 | 新規チケット候補 |
| チケットの description/コメントに「次スプリント」「フォローアップ」作業の言及 | フォローアップチケット候補 |

### 介入の機会

こちらのボールではないが、介入が価値を生む可能性がある場合。

| シグナル | 判断 |
|---------|------|
| 外部ソースの議論が停滞・堂々巡り — 提案がアンブロックに繋がる | 検出対象（disposition で判定） |
| 明確な方向性なく会話が継続 | 検出対象（disposition で判定） |
| 意思決定者不在のまま議論が膠着 | 検出対象（disposition で判定） |

### 構造的欠落

| シグナル | 判断 |
|---------|------|
| 親チケットの description に作業分解があるが子チケットが欠落 | 子チケット候補 |
| 「Xの完了待ち」「X完了後に」だが X に Linear チケットがない | ブロッカーチケット候補 |

## 除外基準

以下は検出**しない**:
- 回答済みで完結した会話
- 議論中だがボールがこちら側にないトピック
- 既に完了した作業
- 既存チケットのスコープでカバーされている作業（重複）

## Disposition

検出された各項目に以下のいずれかを付与:

| Disposition | 意味 | 実行アクション |
|-------------|------|---------------|
| `create` | 新規チケットが必要 | title, description, priority, labels, parent, links, dueDate を指定して作成 |
| `link` | 既存チケットにコンテキストを追加 | 既存チケットにコメント + relation/attachment を追加 |
| `skip` | アクション不要または時期尚早 | アクションなし。Plan に理由を記録 |

### Cleanup「コンテキスト追加」との境界

- **Cleanup コンテキスト追加**: チケットの description/コメントで既に言及されたURLが attachments に未登録 → 登録する
- **Add link**: 新たに発見された外部議論が既存チケットに関連 → コメント + リンクを追加

トリガーが異なる（既存参照 vs 新規発見）が操作は類似。同一分析パスで実行する場合、Cleanup 結果との重複排除で重複を防止する。
